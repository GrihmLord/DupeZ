# DupeZ v5.7.4 — Feature Wire-Up

A deep audit found that seven feature backends shipped in v5.7.0 and v5.7.1 were never connected to anything. They had unit tests, the CHANGELOG listed them as features, but a user could not actually invoke them — the code had zero callers. v5.7.4 wires them up.

## What was disconnected, now working

- **Discord / webhook audit forwarding** — the audit logger now actually fans events out to a configured webhook. Before v5.7.4, setting up a webhook did nothing because the fan-out call was never made.
- **Episode-store rotation** — the 90-day / 5000-file retention policy now runs once per launch. Before, the cleanup function existed but was never called, so the data folder grew forever.
- **OBS overlay server** — auto-starts when enabled, plus a "Toggle OBS Overlay Server" entry in the Tools menu that starts it and shows you the browser-source URL.
- **Risk Score** — Tools → Risk Score… shows the 0-100 score and the six factors that drive it.
- **Diagnostics** — Tools → Diagnostics (F2) runs all eight self-checks and shows what's passing, warning, or failing, with fix hints.
- **Kill Switch panic-stop** — Tools → Kill Switch — Panic Stop, or Ctrl+Alt+X, immediately halts every active disruption.

## Also fixed

- **Help docs no longer describe a removed feature.** A user reported that the Getting Started guide pointed to a "DUPE METHOD" card (Clone, Drop & Pick, Swap, and so on) that isn't in the app. It was real once — the old "Dupe Engine v2" — but it was removed and the help panel was never updated to match. The duplication guide is rewritten to describe how DupeZ actually works today: select the **Red Disconnect** preset and arm a timed cut with the DISCONNECT module sliders (Arm Delay / Duration) or the TIMED DISRUPT button. There is no method card to hunt for — the old "methods" were just different cut timings, which you now set directly.
- **The rest of the in-app help was audited too.** That report prompted a full pass over every help section. Fixed: Voice Control was pointing at a non-existent settings tab (it's in Network Tools → AI / Smart Ops), the GPC panel location was wrong, "Smart Mode" was mislabelled and misplaced, three disruption modules and ten keyboard shortcuts were undocumented. The Help → Hotkeys dialog now builds itself from the live menu so it can never go stale.
- **A guard test now prevents this from recurring.** Every build now cross-checks the in-app help against the real presets, modules, and shortcuts. If the help describes something that doesn't exist, the build fails.
- **Overlay server no longer reports false success.** `OverlayServer.start()` now returns whether the port actually bound, so DupeZ won't claim the OBS overlay started when the port was already in use.
- **New fail-safe release script.** `scripts/release.ps1` replaces the hand-typed release sequence that silently mis-tagged four past releases.

## Still backend-only (honest accounting)

Two features still need real configuration dialogs and are deliberately not shipped half-built:

- **Cut chaining** — needs a multi-stage editor (add/reorder/remove stages, pick a gate per stage).
- **Kill-switch auto-triggers** — the manual panic-stop works now; the automatic triggers (anti-cheat process detection, risk threshold, runaway packet rate) need a settings panel.

Both are tracked for a future release.

## Why this happened

v5.7.0's release notes said "backend ready, UI lands in v5.7.1." v5.7.1 was then re-scoped to a testing/quality pass and the UI wiring was quietly dropped. Nothing caught it because every module compiled and had tests — no check flagged "this backend has no callers." The audit that found it now also added that check to the process.
