# DupeZ v5.7.9

Clumsy parity, controller startup recovery, and fail-closed release integrity.

## What changed

- DupeZ now automatically chooses the actual bundled Clumsy process for
  requests it can represent exactly. It verifies Local/Remote capture layer,
  requested modules, numeric values, and the running Start state before
  hiding the window.
- Native WinDivert remains available for native-only features and only serves
  as an automatic fallback where packet semantics match.
- Lag and Red Disconnect remain pure, separate stages in the automatic
  connection test. Each stage is released before the next begins.
- Duplicate count now means the same thing on both engines: the UI value is
  extra copies, translated to Clumsy's total-packet count.
- Split-mode helper calls use a longer bounded timeout for engine mutations,
  preventing a valid Clumsy startup from being reported as a failure.
- The frozen GPU helper forces in-process architecture before imports,
  eliminating the controller initialization deadlock during recovery.
- A per-user single-instance mutex prevents duplicate GUI processes from
  colliding over persistence locks and the elevated helper pipe.
- Splash startup preserves the real exception and log path, while controller
  recovery failures enter an explicit network-disabled safe mode.
- Dashboard and engine telemetry polling no longer performs helper IPC on the
  Qt UI thread.
- Non-empty process-scoped presets fail closed because WinDivert does not
  expose process IDs at the packet interception layers DupeZ uses.

## Release integrity

- PyInstaller builds are hermetic to `.venv`, explicitly require `PyQt6.sip`,
  and run a frozen-runtime import verifier.
- Previous update-manifest sidecars are removed before each variant build.
- Release preflight verifies the pinned Ed25519 signature and confirms the
  manifest version, installer filename, size, and SHA-256 against the exact
  `DupeZ_Setup.exe`.
- Bundled `clumsy.exe` is byte-identical to the credited
  [Kalirenegade v0.3.4 release](https://github.com/kalirenegade-dev/clumsy/releases/tag/0.3.4).
  The provenance record pins source commit
  `bc87e73066168520d76122c9165e99ea703b166c` and the source archive hash.

## Hardware validation

Hardware tests are deliberately opt-in and require Administrator access plus
an explicitly authorized private-network IP and MAC. They never select the
first scanned device. Native module tests require a real `affected` counter;
a separate test exercises bundled Clumsy GUI/layer/control startup.

Required environment:

```text
DUPEZ_RUN_HARDWARE_SMOKETEST=1
DUPEZ_SMOKETEST_TARGET_IP=192.168.137.42
DUPEZ_SMOKETEST_TARGET_MAC=00:11:22:33:44:55
```

## Upgrade

Install `DupeZ_v5.7.9_Setup.exe`, or use the stable
`DupeZ_Setup.exe` asset from the latest GitHub release after all signing and
artifact preflight gates pass.
