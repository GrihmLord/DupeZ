# DupeZ v5.7.1 — Codebase Quality Pass

No new features. This release is pure code quality: 175 new unit tests across the 10 modules shipped in v5.6.9 + v5.7.0, three real production bugs fixed that the new tests uncovered, episode store rotation to prevent unbounded disk growth, and consolidated architecture documentation.

## Bugs fixed

The audit pass uncovered three latent production bugs that the previous test suite never caught:

1. **Audit webhook rate-limiter dropped first ~1s of events.** Newly-registered audit sinks (Discord webhook, generic webhook) silently discarded every event for the first second after subscription because `_TokenBucket` started with zero tokens. Bucket now starts full.

2. **Custom preset import crashed on duplicate names.** When you imported a `.dupez-preset.json` file with a name already in the store, the auto-rename logic produced `"Name (2)"` — which then failed the name-regex validation because parentheses weren't allowed. Regex updated; import-collision flow now works end-to-end.

3. **Overlay server class-attribute leak between instances.** Two `OverlayServer` instances on different ports would clobber each other's controller reference. Refactored to per-instance handler factory.

## Episode store rotation

`app/data/episodes/` grows unbounded as you fire cuts. Your install has 122 episodes today; over months that becomes MBs then GBs. New `rotate_episodes()` function with sensible defaults:

- **90-day retention** — episodes older than that are deleted regardless of count.
- **5000-file ceiling** — beyond that, oldest-first trim until under the cap.

Both thresholds are configurable. Safe to call from a background thread; never raises.

## Test suite

## ADR-0002

`docs/adr/ADR-0002-key-architectural-decisions.md` consolidates the major decisions baked into DupeZ across v5.5 → v5.7: WiFi self-disrupt default, auto-update fail-closed posture, split-elevation architecture for the GPU map, local-only telemetry policy, plugin trust model, and the rule for when NOT to add a feature.

## What's deferred (documented in CHANGELOG)

The audit found real but lower-priority debt: 396-line and 353-line try blocks that need decomposition, four files over 1500 LOC that need splitting, a hot-path optimization worth an estimated 20-40% throughput, and 673 broad-except callsites worth narrowing. None block shipping; tracked for v5.8.x.
