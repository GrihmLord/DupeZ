# DupeZ v5.7.0 — Telemetry + Safety + Polish

Seven new backend modules, all extensions of existing infrastructure. No new disruption capability — these expose, aggregate, and act on telemetry the engine already produces.

## What's new

**Risk score.** A single 0-100 number with GREEN/AMBER/RED bands derived from your own episode history: recent cut rate, failure streaks, success-rate shortfall, cuts-too-close-together, never-cut ratio, audit-log activity volume. Surfaces "you've been hammering this session" before you notice yourself. Tested on your real episode store and produced 54 (amber) on first call.

**Kill switch.** Configurable safety layer that calls "Stop All Disruptions" when any of N triggers fires. Ships with four trigger types: anti-cheat process detection, risk-score threshold, runaway packet-drop rate, and manual fire (programmatic). Per-trigger cooldown prevents fire loops.

**Diagnostic wizard backend.** Eight self-checks (admin, WinDivert, Npcap, clumsy.exe, update pubkey, data dir writability, Windows Defender posture, episode store) consolidated into a registry with per-check remediation hints. UI dialog wires in next release.

**Discord webhook.** Fan out audit events to a Discord webhook URL. Token-bucket rate limiting (30/min default). IP scrubbing. Generic webhook variant for Slack/Mattermost/Telegram.

**Cut chaining.** Sequence N presets against a target with time / A2S-state / packet-count gates. "Lag 2s → Red Disconnect 5s → verify severed → release" as one click.

**Multi-account quick-switch.** Persistent active-account marker. Tags episode recorder + audit log with which tracker account you're currently playing on.

**OBS overlay endpoint.** Localhost HTTP server (127.0.0.1:4778) exposing current disruption state as JSON + a self-contained HTML page you drop into OBS as a Browser Source.

## What's still backend-only (UI lands in v5.7.1)

The kill switch, diagnostic wizard, Discord webhook config, cut chaining configurator, and OBS overlay control all have working backends in v5.7.0 but no Qt panels yet. They're callable from a Python REPL or from scripts in the meantime. The v5.7.1 release will add menu items + settings dialogs.
