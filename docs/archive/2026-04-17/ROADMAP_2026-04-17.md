## 3. Bugfix shipped this turn: Account tracker CSV/XLSX round-trip

**Symptom reported:** "the account tracker has issues with uploading csv still with pre existing format and tags identifiers."

**Root causes found (4):**

1. `_XLSX_DEFAULT_FIELDS` contained **10** entries but `ACCOUNT_FIELDS` has **11** (`value` was missing between `location` and `status`). When no header row was detected, the positional fallback mapped every column from #4 onward to the wrong field.
2. CSV reader opened with `encoding='utf-8'` ‚Äî Excel-exported "CSV UTF-8" files carry a BOM, which turned the first header into `"\ufeffAccount"`. None of the synonyms matched; the `account` field silently dropped; `_validate_account_data` then rejected every row ("0 imported"). Classic round-trip death.
3. Header-synonym map (`_XLSX_KNOWN_HEADERS`) was thin ‚Äî missed `character`, `character name`, `map`, `server`, `region`, `kit`, `role`, `class`, `inv`, `items`, `primary`, `wishlist`, `tier`, `priority`, `remarks`, `description`, and the user-reported "tag identifiers" (`#email`, `@notes`, etc.).
4. No delimiter sniffing ‚Äî semicolon- or tab-delimited CSVs (common in non‚ÄëUS locales) parsed as one giant column.

**Fix landed in `app/gui/dayz_account_tracker.py`:**
- `_XLSX_DEFAULT_FIELDS = list(ACCOUNT_FIELDS)` ‚Äî single source of truth, off-by-one impossible.
- New `_normalize_header()` helper: lowercases, strips BOM, strips leading `#`/`@`/`:` tag-identifier prefixes, collapses any run of `[\s\-_./\\|]+` to a single space.
- New `_canon_status()` / `_canon_station()`: map case-variant values (`"ready"`, `"READY"`, `"raider kit"`) back to canonical casing so table color chips still render.
- `_process_csv_file()` rewritten: `utf-8-sig` encoding, `csv.Sniffer` delimiter detection (`,`, `;`, `\t`, `|`), positional fallback path for header-less CSVs, unmapped-header report in the success dialog.
- `_process_xlsx_file()` uses the same `_normalize_header()` and canonicalizers.

**Verification:** AST-parsed clean. Inline test harness confirmed all four bug classes fixed. End-to-end round-trip test with BOM + semicolon delimiter + mixed-case headers ingested 2/2 rows correctly (previously: 0/2).
