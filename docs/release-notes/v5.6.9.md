# DupeZ v5.6.9 — Engine Extensions

Four engine extensions, all reusing existing infrastructure. None introduce new disruption capability — they expose what the engine already supports in user-controllable form, plus a one-click data safety net.

## What's new

**Custom preset editor.** Build your own presets beyond Red Disconnect / Lag / God Mode. Tools menu → "Custom Preset Editor". Validate against the method whitelist, save with a name, export to a JSON file you can share, import other people's presets. They appear in the main dropdown below the built-ins.

**Per-port targeting.** Preset params accept a `_ports` list. With it, cuts only touch packets on those TCP/UDP ports. Set DayZ's default ports `2302, 2303, 2304, 2305` and Discord voice / browser / Steam stay running while the game stutters.

**Process-scoped disruption.** Preset params accept `_process_scope`: `"dayz"` (always target DayZ) or `"auto"` (follow whichever DayZ window has focus; pauses on alt-tab). Uses WinDivert's processId filter to only catch packets from `DayZ.exe` / `DayZ_BE.exe` / `DayZ_x64.exe`.

**One-click backup + restore.** Tools → Backup → "Create Backup…". Bundles every persisted file (tracker, settings, presets, episode store, audit log, custom presets) into a single ZIP with a manifest carrying SHA-256 per entry. Restore verifies every entry's hash before writing. Optional DPAPI encryption for backups that travel via Discord/email.

## Test plan

- Open the Preset Editor, create a preset, save it. Confirm it appears in the dropdown.
- Select the custom preset, fire DISRUPT. Check the log for `[PRESET] per-port scope applied` and `[PRESET] process scope applied`.
- Verify Discord voice / browser stay responsive during the cut.
- Create a backup, open the ZIP, confirm `manifest.json` lists every file with a SHA-256.
- Restore the backup in dry-run mode, confirm the preview lists every entry without writing.
