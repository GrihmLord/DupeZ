# DupeZ v5.7.3 — Security Hardening

A focused security review of the eleven modules added between v5.6.9 and v5.7.2. Those modules post-dated the original nation-state certification sweep and had never been security-reviewed. Five findings, one critical — all fixed and locked with regression tests.

## Critical

**Backup restore could execute attacker code.** The "restore from backup" feature wrote every file listed in a bundle's manifest, checked only against escaping the install folder. A maliciously hand-crafted backup — the kind that could be shared in a Discord channel — could contain a `app/core/...py` or `dupez.py` entry with attacker code, which would run the next time DupeZ launched. v5.7.3 restricts restore to `app/data/` and `app/config/` only. Code and packaging files are refused. Genuine backups only ever contain those paths, so nothing legitimate breaks.

## Medium

- **Decompression-bomb protection.** Restore now caps per-file (512 MB) and total bundle (2 GB) expanded size, so a tiny malicious bundle can't exhaust your RAM and disk.
- **OBS overlay no longer leaks to other websites.** The overlay's `/state` endpoint was sending a wildcard CORS header, which meant any website you visited in a browser could quietly read whether DupeZ was running and what it was targeting. That header is gone — the OBS browser source still works (it loads the overlay page directly), but random websites can't poll it anymore.
- **Webhook URLs are scheme-checked.** Audit webhook URLs are now validated to `https://` (or `http://` for localhost only). A `file://` or `ftp://` URL is rejected outright.
- **Shared presets can't hijack the engine.** A preset's params can't carry engine-internal control flags anymore. Only the documented `_ports` and `_process_scope` keys are allowed; anything else is rejected on import. Also added a size cap so a preset file can't be a giant data blob.

## Low

- **Diagnostics fix-commands are display-only.** Documented an explicit contract that the suggested shell commands in the diagnostics panel must never be auto-executed — only shown for you to copy and run yourself.

## Tests

15 new security regression tests (`tests/test_security_v573.py`), one cluster per finding. Total suite: 585 passing.
