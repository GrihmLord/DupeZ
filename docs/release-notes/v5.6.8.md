# DupeZ v5.6.8 — Tracker Save-Bug Fix + Dupe-History Backend

This release fixes a real data-destruction bug in the account tracker and lays the backend groundwork for the v5.6.9 dupe-history panel.

## Important: high-severity bug fixed

Earlier builds (v5.6.7 and prior) had a subtle but destructive flaw in the account-tracker load path. When the tracker loaded zero accounts for any reason — HMAC mismatch during a key rotation, a corrupt file, a transient I/O error — it would seed the starter template (3 rows: "Main Character / Alt — Storage Mule / Alt — Fresh Spawn") AND save those template rows back over your real data. Users who imported a workbook with 11 accounts could launch the next day and find them replaced.

v5.6.8 fixes this by distinguishing "true first launch (no data file on disk)" from "data file exists but loaded as empty." The template only seeds on the former. Anything else now preserves the on-disk file and surfaces an error log so you can investigate before manually re-importing.

If your accounts disappeared on a prior launch, the data may still be recoverable from `app/data/dayz_accounts.backup.*.json` files (the persistence layer keeps timestamped backups).

## Added

- **`LearningLoop.recent_episodes(limit, labeled_only)`** — read-only API returning recent cuts from the existing episode store. Newest-first by start timestamp. Used by the v5.6.9 dupe-history panel that lands next release.
- **`LearningLoop.session_summary()`** — aggregate counts across the full episode store (total / labeled / successes / failures / success_rate / severed / degraded / never_cut / last_session_ts). Suitable for a dashboard header strip.
